哭兮兮的经历

记录了在学习linux系统中遇到的各种坑呀坑

1.昨天在ubuntu服务器上装东西，由于一直在操作/usr文件夹，有时候就在一直改权限，所以一时兴起修改了/usr的权限为777.然后今天再次打开服务器的时候，突然发现sudo命令用不了，权限也就随之改不了，然后在网上了解到可以通过su来再次改变权限。然而难受的是，我在装ubuntu后，没有对su设置密码。所以导致了整个方法也走不通了，然后还看到可以进入recovery模式，进行修改权限恢复，令人再次难受的是，我是云服务器，所以这个方法我也行不通了，最后只有哭兮兮的将系统重装了。

2.之前听说过linux是大小写敏感的，但是具体会是怎么样的，还是没有一个具体的概念，然后一次在运行项目的时候，调用一个接口显示500，然后看原因，提示不存在某一个表。但是数据库中这个表是有的。然后就想到会不会linux中mysql的中相关的也是大小写敏感嘞，然后改写了表名，再次请求，然后成功了。也就是说很多在windows上面不是大小写敏感的东西，有可能在linux上默认都是大小写敏感。当然，对于mysql来说，如果不想它是大小写敏感的的话可以修改mysql的my.cnf文件，在[mysqld]下加入如下代码：
