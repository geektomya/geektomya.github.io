在HTML遇到的坑

发表于 2018-06-02   |   更新于 2019-04-10 | 分类于 Problem
字数统计: 424 | 阅读时长 ≈ 2

简单的学习了网页设计与开发之后，就很少再去看它 。因此在今天写一个带js函数的html的文件时遇到了一个难以理解的问题。

首先看一段代码

刚开始我运行的时候，网页并没有弹出警告框,也就是没有任何的反应，我感觉一脸懵逼，看了几遍代码也没有发现问题。然后突然想起会不会是代码错了，然后在浏览器果断按下了F12，发现

出现Uncaught SyntaxError: Unexpected end of input错误，自行搜索了一下发现，产生这种结果的原因大都是成对的符号没有用正确。突然就醒悟了过来。

在 html 中 ，相邻的两个 成对的符号会自动进行匹配，即testparams前面的双引号与zhanghua前面的双引号进行了匹配。所以导致程序出现了错误。

原来带调用 testparams 的时候 写着 onload=“ ”；这种情况下如果 testparams函数没有参数的话或者将字符串参数写成 单引号 ‘ ’ 的形式都不会出错。当然也可以在调用函数的时候不将函数用 双引号 “ ”形式也可以运行，ps

总结 ：在html里面 如果 有字符串的话尽量写成 单引号 ‘ ‘ 的形式，这样就可以减少出现错误发生的几率。
